# Contributing to University Management System Django

Thank you for your interest in contributing.

## Contributions Are Welcome

Contributions are allowed and welcome through issues, discussions, and pull requests.

Accepted contribution types include:
- Bug fixes
- Security fixes
- Performance improvements
- Documentation improvements
- UI and UX improvements
- Tests and refactoring

## How to Contribute

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make focused and clean changes.
4. Test your changes locally.
5. Open a pull request with a clear description.

## Contribution Rules

- Keep pull requests small and focused.
- Follow existing code style and project structure.
- Do not include unrelated formatting-only changes.
- Add or update tests when behavior changes.
- Use clear commit messages.
